## 1. Overview

Sistema web local para controle de gastos compartilhados no cartão de crédito ou dinheiro emprestado. O foco é a gestão do saldo devedor por pessoa, permitindo registros de dívidas e pagamentos (amortização) que abatem o saldo total.

### 1.1 Goals

- Cadastrar pessoas.
- Registrar transações (Débitos e Créditos/Pagamentos).
- Exibir saldo consolidado por pessoa.
- Suportar saldo credor (quando o pagamento excede a dívida).

### 1.2 Non-Goals

- Integração direta com APIs bancárias.
- Multi-usuário (autenticação complexa).
- Gestão de juros compostos ou parcelamentos automáticos complexos (inicialmente).

## 3. Technical Approach

### 3.1 Architecture Overview

- **Backend:** Node.js com Express para servir a API.
- **Database:** SQLite3 para persistência local em arquivo único.
- **Frontend:** HTML/JS moderno (Vanilla ou React se necessário) servido localmente.

### 3.2 Component Design

- **API REST:** Endpoints para `/people` e `/transactions`.
- **Logic Layer:** Cálculo de saldo em tempo real (Query SQL `SUM` com sinais invertidos).
